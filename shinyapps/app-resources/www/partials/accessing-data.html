Getting access
To access any of the data assets, you should request access through the Data Governance Team. In most cases, access is restricted to analytics data assets. For many data assets, there are extended usage notes which set out the conditions of internal and external use.


Accessing data via an ODBC connection in R
For R users, the simplest way to access the data is via on ODBC connection as per the code snippet below.

# Note the the driver will be "SQL Server" and port will be 1433, as all CI data is currently held on MS-SQL servers.
# You will need to know the names of the Server and Database
#
#
# Establish a connection to the SQL server
con ← DBI::dbConnect(odbc::odbc(),
        Driver = "SQL Server",
        Server = "nameOfTheServer",
        Database = "nameOfTheDB",
        Port = 1433)
#
# Once a connection is established, access the db table using the table name.
#
tableFromDB = tbl(con, "tableInDB")

The DPLYR package can then be used for all data retreival tasks and SQL is not needed.

Accessing data via a REST API
In some cases, R, Python, SAS, STATA, and PowerBI users can access data via Rest APIs. This allows filters on the data to be passed as parameters in a url, and all results are returned JSON format. Note that, during the CIDA project evaluation there will be limited availability for REST API data, as its predominant use case is for the external provision of data. If data is available via a REST API, this will be noted as part of the data asset's metadata


Requesting a copy of a data asset
If it is not possible to access the data through the mechanisms about, you can request a copy of data as a .csv file. Note that, in most cases, you will be required to hold this data on a restricted share drive in the Cancer Institute. If you require data in this format, contact the Data Governance Team.
